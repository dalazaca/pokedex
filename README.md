# Pokedex

Para quien es amante de Pokemón, esta es una Pokédex que permite ver el detalle del Pokemón que selecciones, guardar en favoritos los que más te gusten, compartir información a amigos y por último buscar por nombre. La información fue suministrada gracias a [PokeAPI](https://pokeapi.co/).

El framework utilizado fue [Vue.js](https://vuejs.org/) y se encuentra desplegado con [Github Pages](https://pages.github.com/).

## Tecnologías principales
- [Vite](https://vite.dev/): Acelera el desarrollo con un servidor rápido y compilación optimizada.
- [Vue.js](https://vuejs.org/): Framework progresivo que promueve desarrollo ágil, mantebible y escalable.
- [Bootstrap](https://getbootstrap.com/): Utilizar componentes predefinidos para agilizar el desarrollo, adaptándose a los estilos responsivos y asegurando la compatibilidad con distintos navegadores.
- [Router](https://router.vuejs.org/): Facilitar la navegación al poder crear rutas dinámicas, generando una navegación fluída y guardando el historial, esencial para proyectos de tipo SPA.
- [Pinia](https://pinia.vuejs.org/): Ideal para manejar el estado global en aplicaciones Vue, manteniendo el código organizado y escalable.

## Tecnologías complementarias
- [Husky](https://typicode.github.io/husky/): Con el hook commit-msg, se asegura que los mensajes sigan un formato específico (ej: convenciones como Conventional Commits), mejorando la consistencia y calidad del historial de commits.
- [ESLint](https://eslint.org/) y [Prettier](https://prettier.io/): Mejora la legibilidad del proyecto, reduciendo la decisión de definir la estructura del código y facilitando la colaboración con futuros desarrolladores.
- [i18n](https://vue-i18n.intlify.dev/): Permite adaptar la aplicación a múltiples idiomas si es que en un futuro se desea agregar un nuevo idioma sin necesidad de hacer un refactor masivo.
- [Axios](https://axios-http.com/es/docs/intro): Manejo simple de solicitudes HTTP con soporte para promesas, interceptores y manejo automático de JSON.
- [Github Pages](https://pages.github.com/): Ideal para demostraciones, sin necesidad de servidores externos ni correr el proyecto en local.
